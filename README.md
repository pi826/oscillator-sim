# oscillator-sim

位相振動子のインタラクティブシミュレータ。振動子が住む「状態空間」を
S¹(標準円)/ 距離グラフ(自己交差曲線)/ S²(球面)から選び、
リアルタイムに時間発展・観測量・状態分類を表示します。

## 実行方法

```
uv sync
uv run python -m oscillator_sim
```

依存: Python 3.11+, PySide6, pyqtgraph, numpy, PyOpenGL(すべて `uv sync` で導入)。

### コマンドラインオプションとデスクトップ常駐

初期構成はコマンドラインから指定できます(名前は大文字小文字無視の
部分一致で解決。`--list` で登録名を一覧):

```
uv run python -m oscillator_sim --space graph --curve lissajous -n 80 --speed 20 --play
```

**ビューアモード**(キャンバスのみ・自動再生)でデスクトップに
常駐させる例:

```
# 通常ウィンドウのビューア(Space で一時停止、Esc で終了)
uv run python -m oscillator_sim --space graph --curve lissajous --viewer

# デスクトップウィジェット風: 枠なし・最背面・タスクバー非表示
uv run python -m oscillator_sim --space graph --curve lissajous --desktop --size 600x480 --pos 100,100

# 実験的: デスクトップアイコンの背面(壁紙レイヤー)に全画面埋め込み
# (Windows のみ。操作不可・終了は taskkill /im pythonw.exe など)
uv run python -m oscillator_sim --space graph --curve lissajous --wallpaper
```

**ログイン時の自動起動**は、起動したいコマンドに `--install-startup` を
付けて一度実行すると、スタートアップフォルダに登録されます
(コンソールなしの pythonw で起動する .vbs が作られます):

```
uv run python -m oscillator_sim --space graph --curve lissajous --viewer --install-startup
uv run python -m oscillator_sim --uninstall-startup   # 解除
```

ビューアモードでは観測量・分類の計算(KS 統計量など)を省略して
CPU 負荷を抑えます。`--fps 20` などでさらに描画レートを下げられます。

    ├── main_window.py#   QTimer 駆動。状態空間切替でキャンバス・コントロールを切替
    ├── controls.py   #   n・空間・モデル・曲線・ω・分岐のプルダウン、対数スピード

計算(core)・状態空間(space)・幾何(geometry)・表示(gui)は厳密に分離
されており、gui だけが Qt に依存します。モデル・曲線・分岐規則・結合関数は
レジストリ(`registry.py`)にデコレータで登録するだけで GUI のプルダウンに
現れます。

## 数値計算

- RK4、固定刻み dt = 0.005(`constants.DEFAULT_DT`)。
- 時間発展スピードは「1 フレームあたりの計算ステップ数」(対数スライダー、
  0.1〜1000)で制御され、描画レート(約 30 fps)から分離されています。
- 乱数はシード指定可能な単一の `numpy.random.Generator` で管理され、
  初期配置・ω の生成・分岐規則の乱数すべてが同一シード管理下にあります。

## S¹ モード

- モデル(プルダウン)。パラメータはモデル側の宣言から GUI がスピン
  ボックスを自動生成:
  - **Kuramoto**: `dθᵢ/dt = ωᵢ + (K/n) Σⱼ sin(θⱼ−θᵢ)`
  - **Sakaguchi–Kuramoto**: `sin(θⱼ−θᵢ−α)`
  - **Cotangent (nearest neighbor)**:
    `dθᵢ/dt = ½(cot((θᵢ−θᵢ₋₁)/2) − cot((θᵢ₊₁−θᵢ)/2))`(添字は巡回:
    θₙ₊₁ = θ₀, θ₋₁ = θₙ)。ω 項を持たない厳密な式の実装で、隣接位相を
    両側から押し返す勾配流として splay 状態に緩和します。cot(x/2) は
    x について 2π 周期なので位相差の巻き戻しは値を変えません。
    厳密な場は隣接位相の衝突で発散するため、固定刻み RK4 の安定化の
    ために |cot| を `constants.COT_CLIP`(= 100)でクリップしています
    (順序が保たれた初期値では衝突は起きず、クリップは作用しません)。
  - **Kuramoto–Daido (2 harmonics)**: `(1/n) Σⱼ [K₁ sin Δ + K₂ sin 2Δ]`。
    第 2 調和が優勢だと位相が mod π でロックし 2 クラスタ
    ((m,2)-pattern)を形成。
  - **Winfree**: `dθᵢ/dt = ωᵢ + (K/n) Σⱼ (1+cos θⱼ)·(−sin θᵢ)`。
    平均場位相モデルの原型(パルス×感受性)。
  - **Active rotator**: `dθᵢ/dt = ωᵢ − a sin θᵢ + (K/n) Σⱼ sin(θⱼ−θᵢ)`。
    a > ω で興奮系(excitable)になる。
  - **Kuramoto (nearest neighbor)**: 添字リング上の最近接結合
    `(K/2)[sin(θᵢ₊₁−θᵢ) + sin(θᵢ₋₁−θᵢ)]`。twisted 状態
    θⱼ = 2πmj/n が安定に存在し、(m,q)-pattern 分類で観察できます。
  - **Nonlocal ring (chimera)**(Kuramoto–Battogtokh):
    `dθᵢ/dt = ωᵢ − K Σⱼ Gᵢⱼ sin(θᵢ−θⱼ+α)`、
    Gᵢⱼ ∝ exp(−κ·dᵢⱼ)(dᵢⱼ は添字リング距離 |i−j|/n、行和 1 に正規化)。
    α ≈ 1.45・κ ≈ 4・ω 同一・n を大きめ(100 前後)にすると、同期
    ドメインと乱雑ドメインが共存する**チメラ状態**が観察できます。
    κ = 0, α = 0 では全結合 Kuramoto に厳密に一致(テストで確認)。
  - **Kuramoto (inertia)**(電力網 / swing 方程式型):
    `m θ̈ᵢ + θ̇ᵢ = ωᵢ + (K/n) Σⱼ sin(θⱼ−θᵢ)`。唯一の 2 階系で、
    `CircleDynamics` が拡張状態 (θ, θ̇) を RK4 で積分します。速度は
    θ̇ᵢ(0) = ωᵢ(自由回転)で初期化され、振動子数が変わると再初期化
    されます。慣性 m を大きくすると同期への緩和が振動的になります。
- ω は「全て同一 / 正規分布」(平均・分散は `constants.py`)。
- 分類: 一般化秩序パラメータ r_m = |Σⱼ exp(imθⱼ)|/n (m = 1..n) と
  位相順序の巻き数 w で判定。r_m ≈ 1 となる最小の m を q として
  q = 1 → sync、q = n かつ |w| = 1 → splay、その他の q → (w,q)-pattern、
  どの r_m も 1 に近くない → incoherent。閾値は `constants.R_ONE_THRESHOLD`。
- 小窓に r₁(t) を時系列表示。点の色は位相の色相環。

## グラフモード(自己交差曲線上の自由移動)

S¹ 力学とは別の力学系です。はめ込み γ: S¹ → R² の像を、自己交差点を
次数 4 の頂点とする距離グラフとみなします。

曲線プリセット(S¹ モードでは表示・配置のみ、グラフモードでは
グラフの形状を定義):
- **Unit circle** — 交差なし(頂点 0・辺 1 の閉ループ)
- **Limacon**(r = 0.5 + cos φ)— 交差 1 点(頂点 1・辺 2)
- **Figure eight**(Gerono レムニスケート)— 交差 1 点(頂点 1・辺 2)
- **Trefoil**(ハイポトロコイド R=3, r=1, d=1.5)— 交差 3 点(頂点 3・辺 6)
- **Lissajous (3,2)**(x = sin 3s, y = sin 2s)— 交差 7 点(頂点 7・辺 14)

- **状態**: (辺 e, 辺上の弧長座標 s, 進行方向 σ ∈ {+1, −1})。
- **運動**: 速さ `ωᵢ + (K/n) Σⱼ g(d(xᵢ, xⱼ))` で進行方向に移動。
  結合関数 g(d) は `GraphCoupling` 抽象クラスで差し替え可能(プルダウン):
  - `sin(2πd/L)` — 標準(L は全長)
  - `−sin(2πd/L)` — 符号反転。群れを散らし一様化(U → 1)しやすい
  - `sin(4πd/L)` — 第 2 調和。2 つの塊への分離を誘発
  - `Mexican hat` — ガウス差分
    `g(d) = exp(−(d/0.1L)²) − 0.5·exp(−(d/0.3L)²)`(近距離ブースト+
    中距離ダンピング)。等間隔の「パルス列」への自己組織化を誘発
- **測地距離**: 辺内は弧長差、辺をまたぐ場合は頂点経由の最短路
  (頂点間距離は Floyd–Warshall で前計算・キャッシュ)。
- **分岐規則**(プルダウン):
  1. **一様ランダム** — 進入した枝(辺×端点。即時 U ターンになる枝)を除く
     候補から等確率で選択。乱数はシード管理下。
  2. **直進** — 進入方向の接線に最も近い出枝を選択。
  3. **S¹ 準拠** — 元の曲線パラメータ順に進む(交差を素通り)。滑らかな
     曲線では直進と一致することをテストで確認済み。
- **イベント検出**: RK4 で合成した変位を適用する際、1 ステップ内で頂点に
  到達したら頂点でちょうど切り、分岐後の辺に残距離を正確に持ち越します
  (K = 0 のとき 5000 ステップ・多数回分岐後の弧長進行誤差 ~1e-13 を確認)。
  RK4 の中間段の位置評価は現在の辺内にクランプされます(分岐は確率的な
  ため中間段では発生させない)。
- **観測量**:
  - 一様性 U(t) = 1 − ½ Σₑ |ρₑ − ℓₑ/L|(小窓に時系列表示)
  - ペア測地距離分布と一様分布時の理論分布(モンテカルロ前計算)の
    KS 統計量(パネルに現在値)
  - 辺ごとの方向偏極 pₑ = (#σ=+1 − #σ=−1)/(辺上の総数)。パネル表示に
    加えて辺の色(青 − 灰 − 赤)でも可視化。
- 点の色は進行方向(σ = +1 暖色、σ = −1 寒色)。

## 球面モード(S² 上の Lohe 力学)

- **状態**: 単位ベクトル xᵢ ∈ S² ⊂ R³(形状 (n,3)、各行のノルム 1)。
- **力学**(`SphereModel` 抽象クラス経由で差し替え可能、プルダウン):
  - **Lohe**: `dxᵢ/dt = Ωᵢ xᵢ + (K/n) Σⱼ [xⱼ − ⟨xⱼ, xᵢ⟩ xᵢ]`
  - **Lohe (phase lag)**: 結合項に z 軸回りの回転 R = R_z(α) を挿入した
    `(K/n) Σⱼ [R xⱼ − ⟨R xⱼ, xᵢ⟩ xᵢ]`。Sakaguchi 位相ラグの S² 版で、
    α ≠ 0 では群れが z 軸回りに旋回しながら秩序化します
    (α = 0 で Lohe に厳密に一致することをテストで確認)。
  歪対称行列 Ωᵢ ∈ so(3) は軸性ベクトル wᵢ として保持され Ωᵢx = wᵢ × x。
  生成は「全て零 / 共通軸 + 角速度の正規分布 / 軸も角速度もランダム」。
- **数値積分**: RK4 の各ステップ後に必ず各 xᵢ を正規化し |xᵢ| = 1 を維持
  (長時間積分後の誤差 ~1e-16 をテストで確認)。
- **観測量**:
  - 秩序パラメータ r(t) = ‖(1/n) Σⱼ xⱼ‖ を小窓に時系列表示。
  - 2 次モーメント S = (1/n) Σⱼ xⱼxⱼᵀ の固有値による分類
    (閾値は `constants.py`):
    - **sync**: r ≥ 0.9
    - **uniform**: r ≤ 0.2 かつ ‖S − I/3‖_F ≤ 0.12
    - **ring**(大円リング): r ≤ 0.2 かつ固有値が (1/2, 1/2, 0) に近い
    - **clustered**: 上記以外
  - S² は単連結で巻き数が存在しないため、S¹ の (m,n)-pattern 分類は
    適用しません。
- **描画**: GLViewWidget 上の半透明球 + 点群。マウスドラッグで視点回転。
- **マウス配置**: 一時停止中、クリック位置から視線レイを飛ばし球面との
  手前側交差点に追加。右クリックで画面上最近傍を削除。
  「ランダム配置」は正規乱数ベクトルの正規化による厳密な一様分布。

## 操作

- **再生 / 一時停止 / リセット**、現在時刻 t を常時表示。
- **一時停止中のみ**: 左クリックで振動子追加、右クリックで最近傍を削除。
  追加された振動子の ω(球面では Ω)は選択中の分布からシード管理下で
  生成されます。
- 配置ボタン: 一様 / ランダム(/ splay は S¹ のみ。S¹ では splay 配置
  θⱼ = 2πj/n は一様配置と一致します)。球面はランダム一様配置のみ。
- n・状態空間・モデル・曲線・ω 分布・分岐規則・シードの変更は
  シミュレーションを再構築(リセット)します。K などのモデルパラメータは
  実行中に変更できます。
- 2D 表示の視野は曲線のバウンディングボックスに固定されます(自動レンジは
  無効。点や辺の色更新のたびに視野が再計算されて画面全体が揺れるのを
  防ぐため)。マウスホイール/ドラッグでの手動ズーム・パンは可能で、
  リセットで元の視野に戻ります。
- キーボード: Space = 再生/一時停止、Esc = 終了。

## 拡張の仕方

- **新しい S¹ モデル**: `core/models.py` の `OscillatorModel` を継承し
  `name`・`params` を宣言、`dtheta` を実装して `@MODELS.register`。
- **新しい曲線**: `geometry/curves.py` の `Curve` を継承して
  `@CURVES.register`(自己交差があればグラフモードで自動的に頂点になる)。
- **新しい分岐規則 / 結合関数 / 球面モデル**: それぞれ
  `@BRANCHING_RULES.register` / `@GRAPH_COUPLINGS.register` /
  `@SPHERE_MODELS.register`。
- 計算バックエンド差し替え: `core/integrator.py` の `Stepper` と
  `core/simulation.py` の `Dynamics` が境界です。
